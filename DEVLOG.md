## 2026-06-21 — КРИТ: генерация падала с «operation was aborted» + читаемые логи

### Что
1. **Главный баг:** любая генерация мгновенно (через ~2 мс) падала с
   `generate:stream:openai: network error … This operation was aborted`.
   Соединение/ключ/прокси были 100% рабочие (проверены в SillyTavern) — виноват
   был наш код, а не сеть.
2. Логи запроса теперь **форматируются построчно** (как в ST), а не одной
   нечитаемой простынёй.
3. В лог выводится **сырое тело ошибки от апстрима** и реальная причина сетевой
   ошибки (`err.cause`), а не бесполезное «fetch failed».

### Причина abort (важно)
В Express-шиме (`src/_compat/elysia.ts`) `AbortController` для апстрим-фетча
вешался на `req.on('close', …)`. В современном Node событие `'close'` у
**запроса** срабатывает, как только дочитано тело запроса — для обычного JSON
POST это происходит сразу же. То есть мы сами обрывали upstream-fetch через
пару миллисекунд после старта, на каждом запросе.

### Как починили
- Слушаем `'close'` у **ответа** (`res`), а не запроса. И аборт шлём только
  если ответ ещё не завершён (`!res.writableEnded`) — иначе нормальное
  завершение запроса принималось бы за «клиент отвалился». `buildCtx` теперь
  получает `res`.
- `loggedFetch` (`src/llm/generate.ts`):
  - тело запроса печатается через новый `prettyBody()` —
    `JSON.parse` → `JSON.stringify(…, null, 2)` (многострочно). Не-JSON
    откатывается на старый `preview()`.
  - на `!res.ok` логируем `res.clone().text()` — сырой ответ провайдера
    (реальная причина: неверный ключ, нет модели, HTML-502 от прокси и т.п.).
  - в catch печатаем и `err.message`, и `err.cause` (там undici прячет
    `ECONNREFUSED`, TLS-фейл прокси, DNS, abort).

### Грабли
- `res.clone()` обязателен: тело Response одноразовое, иначе сломали бы
  последующее чтение в `mapHttpError`.
- Это объясняло и «(direct, no proxy)» в логе у пользователя с прокси —
  отдельно проверить, что `outboundProxy` реально проставлен в конфиге; код
  прокси (undici ProxyAgent) корректен, просто abort убивал запрос раньше.

## 2026-06-21 — UX: персонаж без картинки + модели через `/v1/models`


### Что
Две правки по фидбеку:
1. **Создание персонажа без аватара.** В правой панели Characters раньше была
   только кнопка «Import» (выбор файла) — создать пустого персонажа было нельзя
   вообще. Добавили кнопку «New character» (иконка `user-plus`): создаёт пустую
   карточку и сразу открывает её в режиме редактирования. Аватар — опция, а не
   обязательное условие.
2. **Модели тянутся с провайдера, без сохранения соединения.** Бывшая кнопка
   «Test» работала только для уже сохранённого соединения (`/:id/test`) — в
   режиме создания приходилось полагаться на захардкоженный каталог моделей.
   Теперь есть stateless-проба.

### Зачем
Захардкоженные списки моделей бессмысленны: почти все провайдеры —
OpenAI-совместимые, отличаются лишь base_url. Список моделей правильно тянуть с
`GET {base}/v1/models` (или эквивалента у Gemini), а не держать в коде.

### Как
- **Бэкенд** (`src/llm/generate.ts`): выделили из `testConnection(id)` чистую
  `testConnectionConfig(cfg: ProbeConfig)`, работающую на «сырых» полях
  (`kind, base_url, api_key, extra_headers`) без обращения к БД. Старый
  `testConnection` теперь просто читает соединение и зовёт её.
- **Роут** (`src/routes/connections.ts`): `POST /api/connections/probe` —
  принимает поля формы, возвращает `{ ok, models[], error }`. Дёргает
  `testConnectionConfig`.
- **Парсинг ответа** универсальный: OAI/Anthropic `{data:[{id}]}`, Gemini
  `{models:[{name:'models/...'}]}` (срезаем префикс), плюс голый массив.
- **Фронтенд**: `api.connections.probe()` + `connections.probe()` в state.
  Кнопка в `ConnectionForm.svelte` переименована в «Fetch models», убрали
  блокировку `!editing` — теперь работает и в create, и в edit, по тому, что
  набрано в форме (с учётом extra-headers). Результат кладётся в комбобокс
  моделей (live-результат имеет приоритет над каталогом).

### Грабли/решения
- `extra_headers` в БД лежит строкой, а из формы приходит объектом —
  `ProbeConfig.extra_headers` принимает оба варианта и парсит при
  необходимости.
- Хардкод-каталог не удаляли: он остаётся как удобный пресет base_url/kind при
  выборе провайдера. Но он больше не единственный источник моделей.
- Сборка: `npm run typecheck` (бэк) чисто, `web && npm run build` (vite) — ок.
  IDE-ошибки про `$state/$derived` в `*.svelte.ts` — шум tsc-сервера (руны
  резолвит только svelte-плагин), на сборку не влияют.

## 2026-06-21 — Termux: полный лог, фикс прокси, `.nomedia`

### Что
1. **`.nomedia`** в корне репо (через `req.sh`) и в `data/` (на старте сервера).
2. **Централизованный логгер** `src/log.ts` — цветные теги, таймстампы,
   режимы через `TAVERN_LOG` (`verbose`|`trace`|`quiet`).
3. **HTTP-лог** каждого запроса к серверу (`→ POST …`, `← … 200 (123ms)`).
4. **Фикс исходящего прокси** и полный лог LLM-трафика.

### Зачем
- Android MediaScanner принимает `.ts`-исходники за MPEG-TS видео (ролики «0:00»
  в Галерее), а аватары/фоны — за картинки. `.nomedia` отключает сканер для
  папки целиком.
- Нужна была видимость происходящего прямо в Termux: какие запросы, через какой
  прокси, с каким телом и статусом.

### Как / Грабли
- Старый код задавал прокси через `fetch(url, { proxy })` — это **Bun-only**
  опция, на Node (Termux) она молча игнорировалась → `outboundProxy` не
  работал вообще. Переписали на `undici` `ProxyAgent` (чистый JS, ставится на
  Termux без компиляции). Весь upstream-трафик идёт через единый `loggedFetch`,
  который и применяет прокси, и логирует обмен.
- Добавили зависимость `undici@^6`.

## 2026-06-21 — Миграция тестов с `bun:test` на Vitest (на Node)


### Что
Перевели весь backend-набор тестов (8 файлов, 127 тестов) с `bun:test` на
Vitest, запускаемый под стоковым Node с `--experimental-sqlite`. Команда:
`npm test` → `node --experimental-sqlite ./node_modules/vitest/vitest.mjs run`.

### Зачем
Уходим с Bun-зависимости в рантайме (см. предыдущие записи про Termux/TUR):
приложение уже работает на `node:sqlite`, тесты тоже должны гонять на Node без
отдельного тулчейна.

### Как
- Во всех файлах `import { ... } from 'bun:test'` → `from 'vitest'`.
- `tests/sqlite-shim.ts`: `node:sqlite` ещё экспериментальный и отсутствует в
  `module.builtinModules`, поэтому vite-node не считает его билтином и пытается
  *трансформировать* (срезает префикс `node:` → ищет несуществующий пакет
  `sqlite`). Шим тянет модуль через `createRequire`, минуя загрузчик. В
  `vitest.config.ts` навесили `resolve.alias['node:sqlite']` на этот шим —
  только для тестов; прод по-прежнему импортирует билтин напрямую.
- `tests/mock-server.ts`: `Bun.serve({ port: 0 })` отдавал порт синхронно, а у
  Node `server.listen(0)` биндится на следующем тике → `server.address()` ===
  null при синхронном чтении. Теперь сами выбираем случайный высокий порт и
  биндимся на него — контракт `const port = serve(...)` сохранён.
- `tests/png-util.ts` (`makePng`/`pngMeta`): заменили `sharp` (которого нет в
  зависимостях) на ручную генерацию минимального валидного PNG через наш
  `encodeChunks`, и чтение размеров из IHDR.
- Пара тестов делала `require('../../src/core/*.ts')` внутри теста — под
  strip-only загрузчиком Node это падает («parameter property is not supported
  in strip-only mode»). Переписали на обычные ESM-импорты сверху файла.
- `messages.test.ts`: `Set.prototype.intersection` (ES2025) ругался по типам —
  переписали проверку пересечения id через `.some()/.has()`.

### Грабли/решения
- `server.deps.external` / `resolveId`-плагин для `node:sqlite` **не помогли** —
  vite-node всё равно пытался грузить модуль через свой загрузчик. Сработал
  только alias на require-шим.
- В Vitest 3 «висящие» `expect(...).rejects` без `await` станут ошибкой — пока
  только предупреждение, но стоит дойти и проставить `await`.

## 2026-06-20 — Фикс: `Unable to locate package bun` в Termux


### Симптом
`./req.sh` падал на установке: `ERROR Unable to locate package bun`.

### Причина
`bun` нет в основных репозиториях Termux. Он живёт в **TUR** (Termux User
Repository), который по умолчанию не подключён.

### Решение
В `req.sh` перед установкой bun теперь добавляется TUR:
```sh
pkg install -y tur-repo
pkg install -y bun
```
Делается только если `bun` ещё не найден. Подсказку при ручной установке тоже
обновили: `pkg install tur-repo && pkg install bun`.

### Запасной вариант
Если в TUR по какой-то причине нет сборки под конкретную архитектуру/версию —
ставить bun официальным скриптом `curl -fsSL https://bun.sh/install | bash`
(но в Termux предпочтительнее пакет из TUR).

## 2026-06-20 — Фикс: `./req.sh: permission denied` в Termux


### Симптом
После клона на телефоне `./req.sh` падал с `permission denied`.

### Причина
Git на Windows не хранит Unix-флаг исполняемости (`+x`). После клонирования в
Termux скрипты приходили без него, и shell отказывался их запускать.

### Решение
Проставили бит исполняемости прямо в индексе git (кроссплатформенно, не зависит
от Windows-ФС):
```sh
git update-index --chmod=+x req.sh start.sh
```
Режим у обоих файлов сменился `100644 → 100755`, закоммитили и запушили. Теперь
при клонировании скрипты сразу исполняемые.

В README добавили подсказку на случай, если бит всё же потеряется:
`chmod +x req.sh start.sh` или просто `bash req.sh`.

## 2026-06-20 — Адаптация под Termux (телефон) + удаление групповых чатов


### Что сделано
- Проект теперь запускается на Android в Termux.
- Удалены групповые чаты (на бэкенде).
- Добавлены два скрипта: `req.sh` (установка/обновление) и `start.sh` (запуск).
- Добавлен `.gitattributes` для корректных переводов строк в `.sh`.
- Заведён этот дневник (`DEVLOG.md`).

### Зачем
Нужно было гонять сервер с телефона: клонируешь репо в Termux, ставишь
зависимости одним скриптом, запускаешь другим. Групповые чаты не нужны.

### Как
**Termux-совместимость — убрали `sharp`.**
`sharp` — это нативная C-библиотека (libvips) для обработки картинок. В Termux
она практически не собирается и ломала установку. Она использовалась только в
двух местах, поэтому:
- `src/files/avatar.ts` — теперь аватар сохраняется «как есть». Формат
  изображения определяется по «магическим байтам» (PNG/JPG/GIF/WEBP). Кроп и
  ресайз на сервере убраны. **Важно:** сигнатуру функции `normalizeAvatar`
  не меняли (аргумент `crop` остался, но игнорируется), поэтому весь
  вызывающий код — `core/characters.ts`, `routes/personas.ts` — работает без
  правок.
- `src/files/thumbnail.ts` — превью больше не генерируются на сервере, отдаётся
  оригинал картинки (браузер масштабирует через CSS). Функцию `getOrGenerate`
  оставили, чтобы роут `/thumbnails/...` не менялся.
- `src/files/blobs.ts` — валидация: аватары теперь могут быть не только `.png`,
  а любым из png/jpg/webp/gif.
- `package.json` — зависимость `sharp` удалена. Нативных сборок больше нет.

**Удаление групповых чатов.**
На фронтенде их уже не было (в макросах стоял комментарий `groups dead`).
На бэкенде:
- удалили `src/core/groups.ts` и `src/routes/groups.ts`;
- в `src/server.ts` убрали импорт, `.use(groupRoutes)` и тег `groups` из Swagger.
- **Намеренно НЕ трогали** таблицы `groups`/`group_members` в схеме БД
  (`src/db/migrations/001_init.sql`): таблица `chats` имеет `CHECK`-ограничение
  «ровно один из character_id / group_id», и `lorebook_bindings` ссылается на
  группы. Вырывать это из схемы рискованно и могло внести баги. Пустые
  неиспользуемые таблицы безвредны.
- Также НЕ трогали worldinfo-«группы» в лорбуках (`group`, `groupWeight`,
  `groupOverride` в `web/src/lib/core/worldinfo-scan.ts`) — это **другая** фича
  (взаимоисключающие группы записей лорбука), не имеет отношения к групповым
  чатам.

**Скрипты запуска.**
- `req.sh`: на Termux ставит `git bun unzip` через `pkg`; затем `bun install`
  для бэкенда; затем в `web/` — `bun install` + `bun run build`. Vite собирает
  фронтенд сразу в корневой `./dist` (см. `web/vite.config.ts` → `outDir`),
  откуда сервер его и раздаёт (`src/server.ts`). Запускать повторно после
  `git pull` для обновления.
- `start.sh`: проверяет, что `node_modules` и `dist` на месте (иначе просит
  сначала `./req.sh`), потом `bun run src/index.ts`. Лишние аргументы
  пробрасываются: `./start.sh --host 0.0.0.0` — доступ с других устройств в той
  же Wi-Fi (токен генерируется автоматически на первом запуске).

### Грабли и решения
- **CRLF в `.sh`.** Файлы редактируются на Windows. Если закоммитить скрипты с
  «виндовыми» переводами строк (CRLF), bash в Termux падает с `bad
  interpreter: No such file or directory`. Решение — `.gitattributes` с
  правилом `*.sh text eol=lf`.
- **Автоформаттер тасовал пустые строки** при правках `server.ts`/`package.json`
  через replace_in_file. Это косметика, на работу не влияет.
- Команды установки (`bun install`) на машине разработки не запускались по
  просьбе пользователя — проверка сборки осталась на стороне Termux при первом
  реальном запуске.

### Состояние проекта на момент записи
- **Структура чистая**: в корне только конфиги/манифесты (`package.json`,
  `tsconfig.json`, `bun.lock`, `bunfig.toml`, `.gitignore`, `.gitattributes`,
  `README.md`, `DEVLOG.md`, `req.sh`, `start.sh`). Код разложен по папкам:
  `src/` (бэкенд: card/chub/core/db/files/llm/routes), `web/` (фронтенд),
  `tests/`, `scripts/`. Файлов «вне структуры» нет.
- **Фронтенд под телефон подготовлен** (проверено, не нами добавлено):
  - `web/index.html`: правильный `viewport`
    (`width=device-width, ..., interactive-widget=resizes-content` — корректная
    реакция на экранную клавиатуру).
  - В `web/uno.config.ts` заданы брейкпоинты `phone: 450px`, `mobile: 1000px`.

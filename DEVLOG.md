## 2026-06-20 — Фикс: `./req.sh: permission denied` в Termux

### Симптом
После клона на телефоне `./req.sh` падал с `permission denied`.

### Причина
Git на Windows не хранит Unix-флаг исполняемости (`+x`). После клонирования в
Termux скрипты приходили без него, и shell отказывался их запускать.

### Решение
Проставили бит исполняемости прямо в индексе git (кроссплатформенно, не зависит
от Windows-ФС):
```sh
git update-index --chmod=+x req.sh start.sh
```
Режим у обоих файлов сменился `100644 → 100755`, закоммитили и запушили. Теперь
при клонировании скрипты сразу исполняемые.

В README добавили подсказку на случай, если бит всё же потеряется:
`chmod +x req.sh start.sh` или просто `bash req.sh`.

## 2026-06-20 — Аудит секретов в репозитории


### Что сделано
Проверили, не утекли ли секреты в публичный репозиторий. **Вердикт: чисто.**

### Как проверяли
- `git ls-files` — в трекинге только исходники/конфиги. Нет `.env`, `*.db`,
  `secrets.json`, `config.json`, папки `data/`.
- `git log` / `git ls-remote` — история = один «Initial commit», local == remote.
  В прошлых коммитах ничего спрятанного нет.
- Регексп-поиск по ключам/токенам/`-----BEGIN`/`sk-…`/`AKIA…`/`ghp_…`. Нашлись
  только **фейковые тестовые** ключи в `tests/integration/provider-kinds.test.ts`
  (`sk-ant-test`, `AIza-test`, `sk-ant-mykey`) — не настоящие, безопасны.
- `src/config.ts` — `apiToken` по умолчанию пустой, генерируется в рантайме
  (`nanoid(32)`) только при `--host 0.0.0.0` и пишется в `config.json`, которого
  в репозитории нет.

### Подстелили соломку
Добавили `config.json` в `.gitignore`. Раньше его там не было: если запустить
сервер публично, туда запишется реальный API-токен, и его можно случайно
закоммитить. Теперь git его проигнорирует.

### Если секрет всё-таки утечёт (на будущее)
Удалить файл и закоммитить — НЕ достаточно: он останется в истории. Нужно либо
переписать историю (`git filter-repo` / BFG) и форс-пушить, либо — что
правильнее — **считать утёкший ключ скомпрометированным и перевыпустить его** у
провайдера.

### Решение по созданию репозитория (важно на будущее)
Репозиторий создаём **пустым** — БЕЗ галочек «Add README», «Add .gitignore»,
«Choose a license». Причина: эти файлы уже есть в проекте локально. Если
GitHub создаст свои версии и сделает стартовый коммит, ветки разойдутся и
первый `git push` упадёт с ошибкой «отклонено / histories unrelated» —
придётся чинить через `git pull --rebase` или force-push. С пустым репо просто
пушим готовый код.

```sh
cd <папка проекта>
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/G0MERUS/tavern.git
git push -u origin main
```

## 2026-06-20 — Адаптация под Termux (телефон) + удаление групповых чатов


### Что сделано
- Проект теперь запускается на Android в Termux.
- Удалены групповые чаты (на бэкенде).
- Добавлены два скрипта: `req.sh` (установка/обновление) и `start.sh` (запуск).
- Добавлен `.gitattributes` для корректных переводов строк в `.sh`.
- Заведён этот дневник (`DEVLOG.md`).

### Зачем
Нужно было гонять сервер с телефона: клонируешь репо в Termux, ставишь
зависимости одним скриптом, запускаешь другим. Групповые чаты не нужны.

### Как
**Termux-совместимость — убрали `sharp`.**
`sharp` — это нативная C-библиотека (libvips) для обработки картинок. В Termux
она практически не собирается и ломала установку. Она использовалась только в
двух местах, поэтому:
- `src/files/avatar.ts` — теперь аватар сохраняется «как есть». Формат
  изображения определяется по «магическим байтам» (PNG/JPG/GIF/WEBP). Кроп и
  ресайз на сервере убраны. **Важно:** сигнатуру функции `normalizeAvatar`
  не меняли (аргумент `crop` остался, но игнорируется), поэтому весь
  вызывающий код — `core/characters.ts`, `routes/personas.ts` — работает без
  правок.
- `src/files/thumbnail.ts` — превью больше не генерируются на сервере, отдаётся
  оригинал картинки (браузер масштабирует через CSS). Функцию `getOrGenerate`
  оставили, чтобы роут `/thumbnails/...` не менялся.
- `src/files/blobs.ts` — валидация: аватары теперь могут быть не только `.png`,
  а любым из png/jpg/webp/gif.
- `package.json` — зависимость `sharp` удалена. Нативных сборок больше нет.

**Удаление групповых чатов.**
На фронтенде их уже не было (в макросах стоял комментарий `groups dead`).
На бэкенде:
- удалили `src/core/groups.ts` и `src/routes/groups.ts`;
- в `src/server.ts` убрали импорт, `.use(groupRoutes)` и тег `groups` из Swagger.
- **Намеренно НЕ трогали** таблицы `groups`/`group_members` в схеме БД
  (`src/db/migrations/001_init.sql`): таблица `chats` имеет `CHECK`-ограничение
  «ровно один из character_id / group_id», и `lorebook_bindings` ссылается на
  группы. Вырывать это из схемы рискованно и могло внести баги. Пустые
  неиспользуемые таблицы безвредны.
- Также НЕ трогали worldinfo-«группы» в лорбуках (`group`, `groupWeight`,
  `groupOverride` в `web/src/lib/core/worldinfo-scan.ts`) — это **другая** фича
  (взаимоисключающие группы записей лорбука), не имеет отношения к групповым
  чатам.

**Скрипты запуска.**
- `req.sh`: на Termux ставит `git bun unzip` через `pkg`; затем `bun install`
  для бэкенда; затем в `web/` — `bun install` + `bun run build`. Vite собирает
  фронтенд сразу в корневой `./dist` (см. `web/vite.config.ts` → `outDir`),
  откуда сервер его и раздаёт (`src/server.ts`). Запускать повторно после
  `git pull` для обновления.
- `start.sh`: проверяет, что `node_modules` и `dist` на месте (иначе просит
  сначала `./req.sh`), потом `bun run src/index.ts`. Лишние аргументы
  пробрасываются: `./start.sh --host 0.0.0.0` — доступ с других устройств в той
  же Wi-Fi (токен генерируется автоматически на первом запуске).

### Грабли и решения
- **CRLF в `.sh`.** Файлы редактируются на Windows. Если закоммитить скрипты с
  «виндовыми» переводами строк (CRLF), bash в Termux падает с `bad
  interpreter: No such file or directory`. Решение — `.gitattributes` с
  правилом `*.sh text eol=lf`.
- **Автоформаттер тасовал пустые строки** при правках `server.ts`/`package.json`
  через replace_in_file. Это косметика, на работу не влияет.
- Команды установки (`bun install`) на машине разработки не запускались по
  просьбе пользователя — проверка сборки осталась на стороне Termux при первом
  реальном запуске.

- **Структура чистая**: в корне только конфиги/манифесты (`package.json`,

  `README.md`, `DEVLOG.md`, `req.sh`, `start.sh`). Код разложен по папкам:
  `src/` (бэкенд: card/chub/core/db/files/llm/routes), `web/` (фронтенд),
  `tests/`, `scripts/`. Файлов «вне структуры» нет.
- **Фронтенд под телефон подготовлен** (проверено, не нами добавлено):
  - `web/index.html`: правильный `viewport`
    (`width=device-width, ..., interactive-widget=resizes-content` — корректная
    реакция на экранную клавиатуру).
  - В `web/uno.config.ts` заданы брейкпоинты `phone: 450px`, `mobile: 1000px`.
  - `web/src/App.svelte`: реактивный `viewport.isMobile`
    (`matchMedia('(max-width: 1000px)')`).
  - `web/src/lib/components/ui/Drawer.svelte`: на мобиле выключается
    drag-resize, ширина шире.
  - `web/src/app.css`: медиа-правила — на ≤1000px чат во всю ширину
    (`--sheldWidth: 100vw`), на ≤450px уменьшаются аватары.
  - Вывод: каркас адаптивный. Возможные точечные доработки UX под маленький
    экран — отдельной задачей по мере тестирования на реальном телефоне.
